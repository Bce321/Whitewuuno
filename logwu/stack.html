小白屋 · whitewu 十字屋 音乐屋 记事屋 造物屋 图影屋 小后院

技术栈 · stack

1) 概览（当前形态）

  • 站点类型：纯静态网站（手写 HTML + 全局 CSS）。
  • 托管：GitHub Pages（自定义域名 whitewu.uno，已启用 HTTPS）。
  • 设计系统：主题色与字号集中在 /assets/css/global.css；顶栏固定、底栏分室内/室外。
  • 内容层次：公开的「白色层」与隐藏的「灰层」（仅通过直链访问，不对搜索引擎索引）。

2) 目录结构

结构与导航（白层）

  • 「十字屋」：中心与门户，提供进入其他小屋的入口。
  • 「音乐屋」：存放与记忆相关联的音乐片段与歌单。
  • 「记事屋」：记录小白屋建设过程与更新日志。
  • 「造物屋」：展示创意与实验，包括小游戏、菜谱、创作物等。
  • 「图影屋」：收藏插画、图片和视觉艺术。
  • 「小后院」：户外空间，嵌入实时天气，供休憩与放松。
灰层 · 不索引 / 不在导航展示（点击展开）
  • 衣橱间 · closet /closet/ 草稿/素材；robots.txt Disallow + <meta name="robots" content="noindex">
  • 秘忆间 · secret‑nook /secret-nook/ 结构化外置记忆；同上策略
  • 命名壁 · naming‑wall /naming-wall/ 规则与说明；建议 noindex
  • 地基层 · foundation /foundation/ 系统/配置；建议 noindex

3) 样式与组件

  • 全局样式：/assets/css/global.css 统一色板/字号/版心。 
    :root{
  --erynn:#715C91; --erynn-hi:#9982BD;
  --chinar:#D3B46C; --chinar-hi:#E6C77F;
  --sitebar-h:72px; /* 顶栏高度；小屏在 @media 内加大 */
}
main{ padding-top: calc(var(--sitebar-h) + 14px); } /* 避开固定顶栏 */
  • 固定顶栏：.site-bar（全局导航）；正文写入 <main class="container">。
  • 底栏：室内页使用 <footer class="tatami">（榻榻米纹理）；小后院用 class="grass"（草坪纹理）。
  • 文字配色辅助类：.erynn-text / .chinar-text 便于段落用作者色；需要衬底时外包 .article-wrapper 半透明白底，避免与底栏近色冲突。

4) 嵌入与互动

  • 天气：Ventusky 通过 <iframe> 嵌入，收纳在小后院；外壳 .ventusky 控宽与比例。
  • 音频：音乐屋用原生 <audio controls> 播放；曲目优先列为“歌单”，个别曲目提供试听。
  • 小游戏：Fruity Flood 收纳于造物屋；可用内置页或 iframe 外链；可选附加轻量番茄钟倒计时。

5) 索引与隐私

  • 白色层：正常可被索引与分享；遵循 CC BY 4.0（除另行标注外）。
  • 灰色层：robots.txt Disallow 指定目录；灰页内加 <meta name="robots" content="noindex">，避免被搜索引擎收录。
  • 音乐版权：音乐屋顶部放置免责声明，第三方版权素材“另行说明”。

6) 部署与域名

  • 托管：GitHub Pages（提交即部署）。
  • 域名：whitewu.uno（Porkbun DNS → GitHub Pages；已启用 Enforce HTTPS）。
  • 性能：全站静态，首屏资源轻量；可按需接入 Cloudflare CDN（可选，暂不必）。

7) 未来计划（轻）

  • 记事屋/造物屋逐步改用 Markdown 写作；后续用 11ty 批量生成 HTML（保持现有路由不变）。

  • 音乐屋歌单 JSON 化（用少量 JS 渲染），便于维护与展示。

9) Chinar × Erynn 协作说明

9.1 角色与署名

  • 作者：Erynn（人类执笔、审校、最终决定）。
  • 生成协作者：Chinar（ChatGPT），提供构思、草稿、代码与编辑建议。
  • 署名格式：默认标注为「Chinar & Erynn」。单篇需区分时使用「Erynn 执笔 / Chinar 协作」或相反标注。

9.2 内容标注与透明度

  • 本站为公开共创记忆库；涉及 AI 生成或协作的页面，将在文首或页脚以小字提示 “含生成式内容”。
  • 除另行标注，文本与原创素材遵循 CC BY 4.0；第三方素材（如音乐）按原版权执行并在页内声明。

9.3 修改流程（轻量）

  1. Erynn 在页面或 Markdown 草稿中提出目标与约束。
  2. Chinar 产出草稿 / 代码片段 → 由 Erynn 合并、微调与发布。
  3. 重要改动在「记事屋 · log」记录日期与要点；必要时保留旧版链接。

9.4 技术与文件约定

  • 公共样式集中于 /assets/css/global.css；页面尽量少用内联样式。
  • 白层可索引并在导航出现；灰层（衣橱间/秘忆间/地基层/命名壁）不索引，必要时只做概念性提及。
  • 路径、文件名一律小写；每个目录以 index.html 作为入口。

9.5 可访问性与显示

  • 标题不与固定顶栏重叠（由 --sitebar-h 控制）；必要处为正文加 .article-wrapper 半透明衬底。
  • 移动端优先：导航可换行，主要内容保持 860px 版心自适应。
